    Supabase Storage RLS 權限修復指南

    若出現 row-level security policy 錯誤，表示您的 Supabase 尚未開放 Storage Bucket 的寫入與讀取權限。請執行以下設定： 

    -- 1. 確保 daily-files bucket 為公開 (Public)
INSERT INTO storage.buckets (id, name, public)
VALUES ('daily-files', 'daily-files', true)
ON CONFLICT (id) DO UPDATE SET public = true;

-- 2. 允許所有人讀取、上傳、更新與刪除檔案
CREATE POLICY "Public Select daily-files" ON storage.objects
  FOR SELECT USING (bucket_id = 'daily-files');

CREATE POLICY "Public Insert daily-files" ON storage.objects
  FOR INSERT WITH CHECK (bucket_id = 'daily-files');

CREATE POLICY "Public Update daily-files" ON storage.objects
  FOR UPDATE USING (bucket_id = 'daily-files');

CREATE POLICY "Public Delete daily-files" ON storage.objects
  FOR DELETE USING (bucket_id = 'daily-files');

    操作步驟：

    1. 登入您的 Supabase 控制台。
    2. 選擇專案後進入左側選單的 SQL Editor。
    3. 點擊 New Query，貼上上方 SQL 語法並點選 Run 執行。
    4. 回到此頁面重新上傳即可成功！
